# project - usage

Чарт создаёт проект платформы Alauda (`Project`, `auth.alauda.io/v1`) и раздаёт ему
квоты ресурсов по кластерам. Проект - это верхнеуровневая единица, внутри которой
затем создаются namespace'ы и распределяются ресурсы.

| Что создаётся | Когда                                          |
|---------------|------------------------------------------------|
| `Project`     | всегда (когда чарт включён): имя, отображаемое имя, описание и список кластеров с квотами |

Ресурс создаётся только когда чарт включён - это служебный флаг, его проставляет
консоль при заказе.

## Что нужно указать

| Поле                  | Обязательно | Что это                                                              |
|-----------------------|-------------|----------------------------------------------------------------------|
| `project.name`        | да          | Имя проекта: строчные буквы/цифры и дефис (RFC 1123), до 63 символов  |
| `project.displayName` | нет         | Человекочитаемое имя проекта                                          |
| `project.description` | нет         | Описание проекта                                                      |
| `clusters`            | да          | Список кластеров с квотами (минимум один)                            |
| `clusters[].name`     | да          | Имя кластера Alauda                                                   |
| `clusters[].quota`    | нет         | Квоты ресурсов: `limits.cpu`, `limits.memory`, `requests.storage`, ... |

## Именование

Чарт использует cpaas-нейминг (как `managed-namespace`), а не общую 5-частную схему
имён репозитория. Имя ресурса `Project` берётся напрямую из `project.name`; это же
значение проставляется в label `cpaas.io/project`. Значение приводится к нижнему
регистру и обрезается до 63 символов.

## Квоты

`clusters[].quota` - это произвольный набор ключей `ResourceQuota` со значениями в
формате Kubernetes Quantity (строки):

- `limits.cpu` - лимит CPU (`"1"`, `"1000m"`);
- `limits.memory` - лимит памяти (`"2Gi"`, `"2048Mi"`);
- `requests.storage` - лимит на хранилище (`"3Gi"`).

Список ключей не ограничен - можно добавить `requests.cpu`, `requests.memory` и
другие ключи `ResourceQuota` по мере необходимости.

## Валидации

Рендер падает с понятным сообщением, если:

- `project.name` не задан или не соответствует DNS-формату RFC 1123;
- список `clusters` пуст;
- у элемента `clusters[]` не задано `name`.

## Запуск

```sh
helm lint .
helm template release . -f values.minimal.yaml
helm install  release . -f values.minimal.yaml
```

Полный список параметров с описанием - в `values.full.yaml`, готовый рабочий
пример - в `values.minimal.yaml`.
