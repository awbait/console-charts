# managed-namespace - usage

Чарт создаёт изолированное рабочее пространство в кластере: управляемый платформой
`Namespace`, квоты ресурсов (`ResourceQuota`) и, при необходимости, выделенную сеть
(kube-ovn `Subnet`).

| Что создаётся   | Когда                                       |
|-----------------|---------------------------------------------|
| `Namespace`     | всегда (когда чарт включён)                  |
| `ResourceQuota` | всегда: лимиты CPU / память / поды / хранилища |
| `Subnet`        | если включена выделенная подсеть             |

Ресурсы создаются только когда чарт включён - это служебный флаг, его проставляет
консоль при заказе.

## Что нужно указать

| Поле                      | Обязательно | Что это                                                   |
|---------------------------|-------------|-----------------------------------------------------------|
| `namespace.namespaceName` | да          | Имя namespace, формат `<продукт>-<среда>-ns-<назначение>` |
| `namespace.projectName`   | да          | Проект, которому принадлежит namespace                    |
| `namespace.clusterName`   | нет         | Кластер Alauda (по умолчанию `common`)                    |
| `namespace.displayName`   | нет         | Человекочитаемое имя namespace                            |
| `resourceQuotas.limits`   | да          | Лимиты CPU и памяти (напр. `cpu: "4"`, `memory: "8Gi"`)   |
| `resourceQuotas.requests` | нет         | Запросы CPU/памяти (по умолчанию берутся из лимитов)      |
| `resourceQuotas.storage`  | нет         | Квоты на хранилища по StorageClass                        |
| `subnet`                  | нет         | Выделенная подсеть (нужен зарезервированный в Netbox CIDR) |

## Подсеть

Подсеть стоит включать, только если CIDR-блок заранее зарезервирован в Netbox. IP
шлюза вычисляется из CIDR автоматически, подсеть создаётся изолированной (private).
Остальные параметры подсети фиксированы платформой и не настраиваются.

## Запуск

```sh
helm lint .
helm template release . -f values.minimal.yaml
helm install  release . -f values.minimal.yaml
```

Полный список параметров с описанием - в `values.full.yaml`, готовый рабочий
пример - в `values.minimal.yaml`.
